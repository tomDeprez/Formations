  --data-binary "@esql-dataset.ndjson"

Méthode 2 — Logstash

💡
Quand utiliser Logstash ?

Quand vous voulez transformer les données pendant l'import (enrichir avec GeoIP, Grok, renommer des champs) ou importer un CSV/JSON brut. Pour ce dataset déjà formaté, curl reste plus simple.

1

Placer le fichier dans un dossier lisible par Logstash

TerminalBASH
# Linux / Mac
cp esql-dataset.ndjson /tmp/esql-dataset.ndjson
chmod 644 /tmp/esql-dataset.ndjson

# Docker — monter le fichier dans le container
docker run --rm \
  -v /chemin/local/esql-dataset.ndjson:/data/esql-dataset.ndjson \
  logstash:8.x ...
2

Créer le fichier pipeline import-esql.conf

import-esql.confLOGSTASH
input {
  file {
    path           => "/tmp/esql-dataset.ndjson"
    start_position => "beginning"   # lire depuis le début
    sincedb_path   => "/dev/null"    # ne pas mémoriser la position
    codec          => "json"         # chaque ligne est du JSON
  }
}

filter {
  # Sauter les lignes de métadonnées Bulk (contiennent {"index":...})
  if [index] {
    drop { }
  }
}

output {
  elasticsearch {
    hosts => ["http://localhost:9200"]
    index => "%{[_index]}"   # utilise le champ _index du document
    # user     => "elastic"     # décommenter si auth activée
    # password => "PASSWORD"
  }
  stdout { codec => dots }   # affiche un point par document importé
}
3

Lancer Logstash avec ce pipeline

Lancer l'importBASH
# Linux standard
/usr/share/logstash/bin/logstash -f import-esql.conf

# macOS avec Homebrew
logstash -f import-esql.conf

# Docker one-shot
docker run --rm \
  -v $(pwd)/import-esql.conf:/usr/share/logstash/pipeline/logstash.conf \
  -v /tmp/esql-dataset.ndjson:/tmp/esql-dataset.ndjson \
  docker.elastic.co/logstash/logstash:8.11.0

4 La syntaxe pipe en un clin d'œil

FROM esql-products
Source
|
WHERE price > 100
Filtre
|
STATS avg = AVG(price)
Agrégation
|
SORT avg DESC
Tri
|
LIMIT 10
Limite

Chaque | passe le résultat au pipe suivant. L'ordre est toujours FROM → pipes. Chaque pipe transforme ou filtre les données précédentes.

⚠️
Règle d'or ES|QL

ES|QL opère sur des lignes de données, pas des documents imbriqués. Les champs avec des points (product.price) sont des chemins vers des valeurs scalaires. Les tableaux (order.items) nécessitent MV_EXPAND pour être traités ligne par ligne.

⬡ NIVEAU 1 — Débutant

FROM & LIMIT
Vos premières requêtes

Apprendre à lire des données depuis un index, sélectionner un sous-ensemble de résultats et comprendre la structure de base d'ES|QL.

📖 Concepts clés
FROM — obligatoire, toujours en premier. Spécifie l'index source. Supporte les wildcards (esql-*).
LIMIT — optionnel, limite le nombre de lignes retournées. Sans LIMIT, ES|QL retourne 1000 lignes maximum par défaut.
METADATA — ajoute des champs système comme _id et _index à votre résultat.
Syntaxe FROMES|QL
// Un seul index
FROM esql-products

// Plusieurs index
FROM esql-products, esql-orders

// Pattern wildcard
FROM esql-*

// Avec métadonnées système
FROM esql-products METADATA _id, _index

// FROM + LIMIT (le plus courant pour explorer)
FROM esql-products
  | LIMIT 5
💡
Conseil : toujours commencer par LIMIT

Quand vous explorez un nouvel index, ajoutez toujours | LIMIT 10 pour voir la structure des données sans charger tout l'index. Examinez les noms des champs et leurs valeurs avant d'écrire votre vraie requête.

⬡ NIVEAU 2 — Débutant

WHERE — Filtrer
vos résultats

Apprendre à filtrer les données avec des conditions simples et composées : égalité, comparaison, LIKE, IN, IS NULL, AND, OR, NOT.

📖 Opérateurs WHERE
==, != — égalité/différence exacte  |  >, <, >=, <= — comparaison numérique
LIKE "pattern" — wildcard * (n chars) et ? (1 char)  |  RLIKE "regex" — expression régulière
IN (v1, v2, ...) — appartient à la liste  |  IS NULL / IS NOT NULL — champ absent/présent
AND, OR, NOT — logique booléenne, groupement avec parenthèses
WHERE — Exemples de baseES|QL
// Égalité
FROM esql-products | WHERE product.category == "Électronique"

// Comparaison numérique
FROM esql-products | WHERE product.price > 500

// Plage
FROM esql-products | WHERE product.price >= 100 AND product.price <= 500

// LIKE — wildcard
FROM esql-products | WHERE product.name LIKE "*Pro*"

// IN — liste
FROM esql-orders | WHERE customer.country IN ("France", "Belgique")

// IS NULL
FROM esql-products | WHERE product.brand IS NULL

// NOT
FROM esql-orders | WHERE order.status != "cancelled"

// Combinaison complexe
FROM esql-products
  | WHERE product.category == "Électronique"
      AND product.price > 300
      AND NOT product.name LIKE "*Samsung*"
  | LIMIT 10

⬡ NIVEAU 3 — Débutant+

KEEP, DROP, RENAME
Gérer les colonnes

Sélectionner uniquement les colonnes nécessaires, en supprimer, les renommer pour des résultats propres et lisibles.

📖 Concepts clés
KEEP — conserve uniquement les colonnes listées (comme SELECT en SQL)
DROP — supprime les colonnes listées, garde toutes les autres
RENAME a AS b — renomme une colonne (plusieurs possibles : RENAME a AS b, c AS d)
Les wildcards fonctionnent dans KEEP et DROP : KEEP product.*
KEEP, DROP, RENAME — exemplesES|QL
// KEEP — sélectionner des colonnes spécifiques
FROM esql-products
  | KEEP product.name, product.category, product.price
  | LIMIT 5

// KEEP avec wildcard
FROM esql-products
  | KEEP product.*   // tous les champs "product."
  | LIMIT 5

// DROP — supprimer des colonnes
FROM esql-products
  | DROP product.cost, product.weight_g, @timestamp
  | LIMIT 5

// RENAME — renommer pour plus de clarté
FROM esql-products
  | RENAME product.name AS nom_produit,
           product.price AS prix_eur,
           product.category AS categorie
  | KEEP nom_produit, categorie, prix_eur
  | LIMIT 5

⬡ NIVEAU 4 — Intermédiaire

SORT — Trier
vos résultats

Trier sur un ou plusieurs champs, dans l'ordre croissant ou décroissant, gérer les valeurs nulles dans le tri.

📖 Concepts clés
SORT field ASC — croissant (défaut si non précisé)
SORT field DESC — décroissant
SORT a ASC, b DESC — tri multiple (priorité à gauche)
NULLS FIRST / NULLS LAST — contrôle la position des valeurs nulles dans le tri
SORT — exemplesES|QL
// Tri simple croissant
FROM esql-products
  | SORT product.price ASC
  | LIMIT 5

// Tri décroissant — les plus chers en premier
FROM esql-products
  | SORT product.price DESC
  | LIMIT 5

// Tri multiple : par catégorie puis par prix décroissant
FROM esql-products
  | SORT product.category ASC, product.price DESC
  | KEEP product.category, product.name, product.price
  | LIMIT 20

// NULLS LAST — mettre les null à la fin
FROM esql-employees
  | SORT employee.salary_eur DESC NULLS LAST
  | LIMIT 10

// Top 3 produits les mieux notés par catégorie
FROM esql-products
  | SORT product.rating DESC
  | KEEP product.name, product.category, product.rating
  | LIMIT 3

⬡ NIVEAU 5 — Intermédiaire

STATS — Agréger
et analyser

Le pipe le plus puissant. Calculer des statistiques : comptes, moyennes, sommes, min/max, et grouper par valeurs.

📖 Fonctions d'agrégation STATS
COUNT(*) — nombre de lignes  |  COUNT(field) — non-null  |  COUNT_DISTINCT(field) — valeurs uniques
SUM(field) — somme  |  AVG(field) — moyenne  |  MIN/MAX(field) — extremes
MEDIAN(field) — médiane  |  PERCENTILE(field, n) — ex: PERCENTILE(price, 95)
VALUES(field) — liste toutes les valeurs distinctes du groupe
BY field1, field2 — grouper (facultatif — sans BY → agrégation globale)
STATS — du simple au complexeES|QL
// Compter tous les produits
FROM esql-products
  | STATS total = COUNT(*)

// Compter + moyenne par catégorie
FROM esql-products
  | STATS
      nb_produits  = COUNT(*),
      prix_moyen   = AVG(product.price),
      prix_max     = MAX(product.price),
      stock_total  = SUM(product.stock)
      BY product.category
  | SORT prix_moyen DESC

// Nombre de clients uniques par pays
FROM esql-orders
  | STATS
      nb_commandes      = COUNT(*),
      clients_uniques   = COUNT_DISTINCT(customer.id),
      ca_total          = SUM(order.total_eur),
      panier_moyen      = AVG(order.total_eur)
      BY customer.country
  | SORT ca_total DESC

// Taux d'erreur par service
FROM esql-logs
  | STATS
      nb_total  = COUNT(*),
      nb_errors = COUNT(CASE(log.level IN ("ERROR", "CRITICAL"), 1, null)),
      duree_moy = AVG(http.duration_ms)
      BY log.service
  | SORT nb_errors DESC

⬡ NIVEAU 6 — Avancé

EVAL — Créer
des colonnes calculées

Créer de nouveaux champs à partir de calculs, de fonctions mathématiques, de manipulations de chaînes et de dates.

📖 Concepts clés EVAL
EVAL crée ou remplace une colonne avec une expression.
Opérateurs : +, -, *, / — math standard
ROUND(n, decimales) — arrondi  |  ABS(n) — valeur absolue  |  FLOOR/CEIL(n)
LENGTH(str) — longueur  |  UPPER/LOWER(str)  |  CONCAT(a, b) — concaténer
DATE_TRUNC(1 day, field) — tronquer une date  |  DATE_EXTRACT("HOUR_OF_DAY", field)
On peut chaîner plusieurs EVAL ou les combiner dans un seul.
EVAL — exemples progressifsES|QL
// Calcul de marge en valeur absolue
FROM esql-products
  | EVAL
      marge_eur  = product.price - product.cost,
      marge_pct  = ROUND((product.price - product.cost) / product.price * 100, 1)
  | KEEP product.name, product.price, marge_eur, marge_pct
  | SORT marge_pct DESC
  | LIMIT 10

// Valeur totale du stock
FROM esql-products
  | EVAL valeur_stock = ROUND(product.cost * product.stock, 2)
  | KEEP product.name, product.stock, product.cost, valeur_stock
  | SORT valeur_stock DESC
  | LIMIT 10

// Manipulation de chaînes
FROM esql-employees
  | EVAL
      nom_complet    = CONCAT(employee.first_name, " ", employee.last_name),
      anciennete_ans = 2024 - employee.hire_year,
      salaire_mensuel= ROUND(employee.salary_eur / 12, 0)
  | KEEP nom_complet, employee.department,
         anciennete_ans, salaire_mensuel
  | SORT anciennete_ans DESC
  | LIMIT 10

// Extraction d'heure depuis timestamp
FROM esql-orders
  | EVAL
      heure_commande = DATE_EXTRACT("HOUR_OF_DAY", order.date),
      jour_semaine   = DATE_EXTRACT("DAY_OF_WEEK", order.date)
  | STATS nb = COUNT(*) BY heure_commande
  | SORT heure_commande ASC

⬡ NIVEAU 7 — Avancé

CASE — Logique
conditionnelle

CASE permet d'ajouter de la logique conditionnelle puissante — classification, étiquetage, scoring — directement dans votre requête ES|QL.

📖 Syntaxe CASE
CASE(condition1, valeur1, condition2, valeur2, ..., valeur_defaut)
La première condition vraie gagne. La dernière valeur sans condition est le défaut (sinon null).
CASE peut être utilisé dans EVAL et dans STATS (pour compter sous conditions).
CASE — du simple au complexeES|QL
// Classification prix
FROM esql-products
  | EVAL gamme = CASE(

⬡ NIVEAU 9 — Expert

GROK & DISSECT
Parser du texte brut

Parser des champs texte non structurés directement dans ES|QL avec GROK (regex patterns) et DISSECT (délimiteurs fixes).

📖 GROK vs DISSECT
DISSECT — parsing par délimiteurs fixes. Rapide, simple. Idéal quand le format est prévisible.
DISSECT field "%{a} %{b} %{c}"

GROK — parsing avec patterns nommés ES|QL (sous-ensemble des patterns Logstash).
GROK field "%{IP:src_ip} - %{WORD:method}"
Patterns : %{IP}, %{WORD}, %{NUMBER}, %{GREEDYDATA}, %{NOTSPACE}
GROK & DISSECT — exemplesES|QL
// ─── DISSECT — format fixe ────────────────────────────────
// Parser "service: message" depuis log.message
FROM esql-logs
  | WHERE log.message LIKE "*:*"
  | DISSECT log.message "%{mot_cle}: %{detail}"
  | KEEP log.level, mot_cle, detail
  | LIMIT 10

// ─── GROK — patterns nommés ──────────────────────────────
// Parser l'IP client depuis un champ texte
FROM esql-logs
  | GROK client.ip "%{IP:extracted_ip}"
  | STATS nb = COUNT(*) BY extracted_ip
  | SORT nb DESC
  | LIMIT 10

// GROK sur log.message — extraire type d'erreur
FROM esql-logs
  | WHERE log.level IN ("ERROR", "CRITICAL")
  | GROK log.message "%{GREEDYDATA:type_erreur}: %{GREEDYDATA:detail}"
  | STATS nb = COUNT(*) BY type_erreur
  | SORT nb DESC
  | LIMIT 10

// DISSECT + STATS — analyser les patterns de messages
FROM esql-logs
  | WHERE log.level == "ERROR"
  | EVAL premier_mot = SUBSTRING(log.message, 0,
                       LENGTH(log.message))
  | STATS nb = COUNT(*) BY log.message
  | SORT nb DESC
  | LIMIT 15

FROM esql-logs

FROM index
Source obligatoire. Wildcards supportés.

Filtre les lignes. Conditions booléennes.

Agrégations avec regroupement optionnel.

Tri sur un ou plusieurs champs.
SORT price DESC, name ASC
LIMIT n
Nombre max de résultats retournés.
LIMIT 100
KEEP fields
Sélectionner des colonnes (comme SELECT).
KEEP name, price, category
DROP fields
Supprimer des colonnes.
DROP @timestamp, _id
RENAME a AS b
Renommer une colonne.
RENAME price AS prix_eur
EVAL expression
Créer ou modifier des colonnes calculées.
EVAL marge = price - cost
GROK field "pattern"
Parser du texte avec des patterns nommés.
GROK ip "%{IP:src}"
DISSECT field "fmt"

Éclater un champ multi-valué en lignes.

Enrichir avec des données d'une politique.
