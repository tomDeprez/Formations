01 Introduction au filtre Grok

Grok est le filtre le plus puissant de Logstash. Il permet de parser des lignes de logs non structurées en champs nommés et typés, en s'appuyant sur une bibliothèque de patterns regex prédéfinis et composables.

💡
Philosophie Grok

Au lieu d'écrire des regex complexes from scratch, Grok vous fournit des patterns nommés réutilisables. %{PATTERN:nom_champ:type} — c'est tout ce dont vous avez besoin.

01 — Syntaxe fondamentale

Syntaxe GrokGROK
# Forme de base — extraction sans type
%{PATTERN_NAME:field_name}

# Forme typée — conversion automatique
%{PATTERN_NAME:field_name:type}
# Types disponibles : int, float, (string par défaut)

# Pattern sans capture (utilisation en position)
%{PATTERN_NAME}

# Regex inline dans Grok
(?<field_name>votre_regex_ici)

# Combinaison patterns + littéraux
%{IP:src_ip} - %{USER:username} %{HTTPDATE:timestamp}

02 — Configuration dans Logstash

logstash.conf — Structure complèteLOGSTASH
# logstash.conf
input {
  file {
    path => "/var/log/syslog"
    start_position => "beginning"
    sincedb_path => "/dev/null"
  }
}

filter {
  grok {
    # Pattern unique
    match => { "message" => "%{SYSLOGTIMESTAMP:timestamp} %{HOSTNAME:host} %{WORD:program}(?:\[%{POSINT:pid:int}\])?: %{GREEDYDATA:log_message}" }

    # Patterns alternatifs (essayés dans l'ordre)
    match => { "message" => [
      "%{COMBINEDAPACHELOG}",
      "%{COMMONAPACHELOG}",
      "%{SYSLOGLINE}"
    ]}

    # Écraser les champs existants
    overwrite => ["message"]

    # Tag si échec du parsing
    tag_on_failure => ["_grokparsefailure", "grok_fail"]

    # Ajouter des patterns custom
    patterns_dir => ["/etc/logstash/patterns"]
  }

  # Conversion de date
  date {
    match => [ "timestamp", "dd/MMM/yyyy:HH:mm:ss Z" ]
    target => "@timestamp"
  }

  # Géolocalisation IP
  geoip {
    source => "src_ip"
    target => "geoip"
  }

  # Supprimer les champs temporaires
  mutate {
    remove_field => ["message", "timestamp"]
    add_tag => ["parsed"]
  }
}

output {
  elasticsearch {
    hosts => ["localhost:9200"]
    index => "logs-%{+YYYY.MM.dd}"
  }
}

04 — Options avancées du filtre Grok

Options avancéesLOGSTASH
grok {
  # Multiple captures du même pattern sur une ligne
  match => { "message" => "%{IP} %{IP:dst_ip}" }

  # keep_empty_captures — garder les champs vides
  keep_empty_captures => true

  # named_captures_only — ignorer les groupes non nommés
  named_captures_only => true

  # break_on_match — stopper après le 1er pattern matché
  break_on_match => true

  # timeout_millis — timeout du parsing
  timeout_millis => 1500
}

# Condition sur le résultat
if "_grokparsefailure" in [tags] {
  mutate {
    add_field => { "parse_error" => "true" }
  }
  drop { }  # ou router vers une DLQ
}

# Condition sur champ parsé
if [http_status] =~ /^[45]/ {
  mutate { add_tag => ["error_response"] }
}
🔮
Performance Grok

Les patterns sont compilés en regex Java. Évitez les patterns greedy (GREEDYDATA) en début de pattern — ils causent beaucoup de backtracking. Préférez placer vos patterns les plus discriminants en premier.

05 — Patterns Apache / Nginx

COMBINEDAPACHELOG décomposéGROK
# Ce que fait COMBINEDAPACHELOG en interne :
%{IPORHOST:clientip} # IP ou hostname du client
%{HTTPDUSER:ident}   # Identité RFC1413
%{HTTPDUSER:auth}    # Utilisateur authentifié
\[%{HTTPDATE:timestamp}\]
"%{WORD:verb} %{NOTSPACE:request} HTTP/%{NUMBER:httpversion}"
%{NUMBER:response:int}    # Status HTTP
(?:%{NUMBER:bytes:int}|-) # Taille réponse
"%{QS:referrer}"           # Referrer
"%{QS:agent}"              # User-Agent

# Exemple d'utilisation :
grok {
  match => { "message" => "%{COMBINEDAPACHELOG}" }
}

06 — Patterns Syslog

SYSLOGLINE décomposéGROK
# Structure syslog standard :
%{SYSLOGTIMESTAMP:timestamp}  # Ex: Jan  5 12:34:56
(?:%{SYSLOGFACILITY} )?        # Facility optionnelle (<14>)
%{IPORHOST:logsource}         # Source du log
%{SYSLOGPROG}                 # Programme[PID]:
%{GREEDYDATA:message}         # Message restant

# Syslog avec RFC 5424 :
%{POSINT:syslog_version}      # Version protocole

%{HOSTNAME:hostname}          # Hostname
%{WORD:app_name}              # Nom application

Parser les logs SSHD — brute force + connexion légitime

Q3
Parser les logs iptables (pattern custom)
Les logs iptables ont un format propriétaire non couvert par les patterns natifs. Créez un pattern custom dans un fichier patterns/iptables.grok nommé IPTABLES_DROP qui extrait : l'interface réseau (IN=), les IPs src/dst, protocole, ports src/dst, TTL, et la MAC source. Testez-le sur la Source 3.
25 pts
💡

Q4

20 pts
💡

  match => { "message" => "%{IPORHOST:src_ip} - (?:%{USER:username}|-) \[%{HTTPDATE:timestamp}\] \"(?:%{WORD:http_method} %{NOTSPACE:uri}(?: HTTP/%{NUMBER:http_version})?|-) \" %{NUMBER:http_status:int} (?:%{NUMBER:bytes:int}|-) %{QS:referrer} %{QS:useragent}" }
  tag_on_failure => ["_grokparsefailure_nginx"]
}

Réponse Q5 — Pipeline multi-source complet

pipeline-complet.confLOGSTASH
input {
  beats { port => 5044 }
}

filter {

  # ─── NGINX ───────────────────────────────────────────
  if [fields][log_type] == "nginx" or [message] =~ /HTTP\/[0-9.]+"/ {
    grok {
      match => { "message" => "%{COMBINEDAPACHELOG}" }
      tag_on_failure => ["_fail_nginx"]
    }
    grok {
      match => { "agent" => "(?sqlmap|nikto|masscan|nmap|python-requests)" }
      tag_on_failure => []
    }
    if [request] =~ /(?i)(union.select|or.1=1|'--|\.\.\/)/ {
      mutate { add_tag => ["sqli_attempt"] }
    }
    mutate { rename => { "clientip" => "src_ip" } }
  }

  # ─── SSHD ────────────────────────────────────────────
  else if [fields][log_type] == "sshd" or [message] =~ /sshd\[/ {
    grok {
      match => { "message" => [
        "%{SYSLOGTIMESTAMP:timestamp} %{SYSLOGHOST:hostname} %{PROG:program}\[%{POSINT:pid:int}\]: Failed %{WORD:auth_method} for (?:invalid user )?%{USERNAME:username} from %{IP:src_ip} port %{POSINT:src_port:int}",
        "%{SYSLOGTIMESTAMP:timestamp} %{SYSLOGHOST:hostname} %{PROG:program}\[%{POSINT:pid:int}\]: Accepted %{WORD:auth_method} for %{USERNAME:username} from %{IP:src_ip} port %{POSINT:src_port:int} %{WORD}: %{WORD:key_type} %{NOTSPACE:key_fingerprint}"
      ]}
    }
    if [message] =~ /Failed/ { mutate { add_tag => ["ssh_bruteforce"] } }
  }

  # ─── IPTABLES ────────────────────────────────────────
  else if [message] =~ /IPTABLES/ {
    grok {
      patterns_dir => ["/etc/logstash/patterns"]
      match => { "message" => "%{IPTABLES_DROP}" }
    }
  }

  # ─── AGENT DÉTECTION ─────────────────────────────────
  else if [message] =~ /rule_id=/ {
    grok {
      match => { "message" => "%{TIMESTAMP_ISO8601:timestamp} \[%{LOGLEVEL:log_level}\] %{GREEDYDATA:kv_data}" }
    }
    kv { source => "kv_data" field_split => " " value_split => "=" }
    mutate { rename => { "src_ip" => "[event][src_ip]" } }
  }

  # ─── NORMALISATION COMMUNE ───────────────────────────
  if [src_ip] {
    geoip { source => "src_ip" target => "geoip" }
    mutate { copy => { "src_ip" => "[source][ip]" } }
  }

  # ─── CONVERSION DATE ─────────────────────────────────
  date {
    match => ["timestamp",
      "dd/MMM/yyyy:HH:mm:ss Z",
      "MMM  d HH:mm:ss",
      "MMM dd HH:mm:ss",
      "ISO8601"
    ]
    target => "@timestamp"
  }

  mutate { remove_field => ["timestamp", "message"] }
}

output {
  # Alertes critiques → index dédié
  if [log_level] == "ALERT" or "sqli_attempt" in [tags] {
    elasticsearch {
      hosts => ["localhost:9200"]
      index => "alerts-%{+YYYY.MM.dd}"
    }
  } else {
    elasticsearch {
      hosts => ["localhost:9200"]
      index => "logs-%{+YYYY.MM.dd}"
    }
  }
  stdout { codec => "rubydebug" }
}
