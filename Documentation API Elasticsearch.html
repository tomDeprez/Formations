Document API
Créer, lire, mettre à jour et supprimer des documents
PUT /{index}/_doc/{id} Indexer (créer/remplacer) un document

Crée ou remplace entièrement un document à l'ID spécifié. Si l'index n'existe pas, il est créé automatiquement.

Exemple
PUT /armes-fr/_doc/1
{
  "nom": "FAMAS F1",
  "calibre": "5.56x45mm NATO",
  "type": "fusil d'assaut",
  "pays": "France",
  "annee": 1978,
  "en_service": true
}
POST /{index}/_doc Indexer avec ID auto-généré

Crée un document avec un identifiant unique généré automatiquement par Elasticsearch.

Exemple
POST /armes-fr/_doc
{
  "nom": "HK416",
  "calibre": "5.56x45mm NATO"
}
GET /{index}/_doc/{id} Récupérer un document par son ID

Retourne le document complet avec ses métadonnées (_index, _id, _version, found…).

Requête
GET /armes-fr/_doc/1
Réponse
{
  "_index": "armes-fr",
  "_id": "1",
  "_version": 1,
  "found": true,
  "_source": {
    "nom": "FAMAS F1",
    "calibre": "5.56x45mm NATO"
  }
}
POST /{index}/_update/{id} Mise à jour partielle d'un document

Met à jour uniquement les champs spécifiés sans réécrire tout le document.

Exemple
POST /armes-fr/_update/1
{
  "doc": {
    "en_service": false,
    "remplace_par": "HK416 A5"
  }
}
DELETE /{index}/_doc/{id} Supprimer un document
Exemple
DELETE /armes-fr/_doc/1
POST /{index}/_bulk Opérations en masse (Bulk API)

Permet d'effectuer plusieurs opérations (index/create/update/delete) en une seule requête HTTP. Format NDJSON.

Indexation en masse
POST /armes-fr/_bulk
{ "index": { "_id": "2" } }
{ "nom": "Beretta 92FS", "type": "pistolet", "calibre": "9mm Parabellum" }
{ "index": { "_id": "3" } }
{ "nom": "PGM Hecate II", "type": "sniper", "calibre": .50 BMG" }
{ "delete": { "_id": "99" } }

Search API
Recherche full-text, filtres, tri, agrégations
GET /{index}/_search Recherche complète (Query DSL)

Point d'entrée principal de la recherche. Supporte le Query DSL complet d'Elasticsearch.

Recherche full-text simple
GET /armes-fr/_search
{
  "query": {
    "match": {
      "nom": "FAMAS"
    }
  }
}
Recherche avec filtres + tri + pagination
GET /armes-fr/_search
{
  "from": 0,
  "size": 10,
  "query": {
    "bool": {
      "must": [
        { "match": { "pays": "France" } }
      ],
      "filter": [
        { "term": { "en_service": true } },
        { "range": { "annee": { "gte": 1970 } } }
      ]
    }
  },
  "sort": [{ "annee": { "order": "desc" } }],
  "_source": ["nom", "calibre", "annee"]
}
Agrégations
GET /armes-fr/_search
{
  "size": 0,
  "aggs": {
    "par_type": {
      "terms": { "field": "type.keyword" }
    },
    "annee_moy": {
      "avg": { "field": "annee" }
    }
  }
}
GET /{index}/_count Compter les documents
Exemple
GET /armes-fr/_count
{
  "query": {
    "term": { "type.keyword": "fusil d'assaut" }
  }
}
POST /_msearch Multi-search — Plusieurs requêtes en une

Exécute plusieurs recherches en une seule requête HTTP. Format NDJSON (header + body alternés).

Exemple
POST /_msearch
{ "index": "armes-fr" }
{ "query": { "match": { "type": "sniper" } } }
{ "index": "armes-fr" }
{ "query": { "match": { "type": "pistolet" } } }

EQL — Event Query Language
Requêtes orientées événements et séquences temporelles
💡
EQL est conçu pour la détection de menaces et la corrélation d'événements. Idéal pour les cas de cybersécurité, logs d'audit, SIEM.
GET /{index}/_eql/search Recherche EQL
Détection séquence suspecte
GET /logs-*/_eql/search
{
  "query": """
    sequence with maxspan=5m
      [process where process.name == "cmd.exe"]
      [network where destination.port == 443]
  """,
  "size": 10
}
🧮
ES|QL — Elasticsearch Query Language
Langage de requête tabulaire nouvelle génération (ES 8.11+)
💡
ES|QL utilise une syntaxe pipe inspirée de SQL et Unix. Très puissant pour l'analyse et la transformation de données.
POST /_query Exécuter une requête ES|QL
Exemple ES|QL
POST /_query
{
  "query": """
    FROM armes-fr
    | WHERE en_service == true AND pays == "France"
    | STATS count = COUNT(*) BY type
    | SORT count DESC
    | LIMIT 10
  """
}
🗃️
SQL API
Interroger Elasticsearch avec du SQL standard
POST /_sql Requête SQL sur Elasticsearch
Exemple SQL
POST /_sql
{
  "query": "SELECT nom, calibre, annee FROM \"armes-fr\" WHERE pays = 'France' ORDER BY annee DESC LIMIT 5",
  "format": "json"
}
POST /_sql/translate Traduire SQL → Query DSL

Convertit une requête SQL en Query DSL Elasticsearch équivalent. Utile pour l'apprentissage et le débogage.

Exemple
POST /_sql/translate
{
  "query": "SELECT nom FROM \"armes-fr\" WHERE annee > 2000"
}

Ingest API
Pipelines de transformation des données à l'ingestion
PUT /_ingest/pipeline/{id} Créer un pipeline d'ingestion

Un pipeline peut contenir des processors : set, rename, grok, date, gsub, script, etc.

Pipeline enrichissement arme
PUT /_ingest/pipeline/enrichissement-arme
{
  "description": "Normalise les données d'armes françaises",
  "processors": [
    {
      "uppercase": { "field": "nom" }
    },
    {
      "set": {
        "field": "ingested_at",
        "value": "{{_ingest.timestamp}}"
      }
    },
    {
      "script": {
        "source": "ctx.era = ctx.annee < 1950 ? 'historique' : ctx.annee < 2000 ? 'guerre froide' : 'moderne'"
      }
    }
  ]
}
POST /_ingest/pipeline/{id}/_simulate Simuler un pipeline
Test du pipeline
POST /_ingest/pipeline/enrichissement-arme/_simulate
{
  "docs": [
    { "_source": { "nom": "famas f1", "annee": 1978 } }
  ]
}

Watcher
Alertes automatisées basées sur des conditions
PUT /_watcher/watch/{watch_id} Créer une alerte (watch)

Une watch comprend : trigger (quand), input (données), condition (filtre), actions (email, webhook, index).

Alerte nouvelles armes indexées
PUT /_watcher/watch/alerte-nouvelles-armes
{
  "trigger": {
    "schedule": { "interval": "1h" }
  },
  "input": {
    "search": {
      "request": {
        "indices": ["armes-fr"],
        "body": {
          "query": {
            "range": { "ingested_at": { "gte": "now-1h" } }
          }
        }
      }
    }
  },
  "condition": {
    "compare": { "ctx.payload.hits.total.value": { "gt": 0 } }
  },
  "actions": {
    "notifier_equipe": {
      "webhook": {
        "method": "POST",
        "url": "https://hooks.slack.com/services/XXX",
        "body": "{{ctx.payload.hits.total.value}} nouvelles armes indexées"
      }
    }
  }
}

Cas d'usage complet — Registre Armes FR
Implémentation d'un registre national d'armes avec Elasticsearch
🎯
Ce cas pratique illustre l'utilisation combinée des principales API Elasticsearch pour construire un registre d'inventaire d'armes françaises : indexation, recherche avancée, agrégations, et alertes.
🏗️ Architecture du projet
Un registre national d'armes nécessite : stockage typé, recherche multi-critères, statistiques par catégorie, cycle de vie des données, et alertes sur nouvelles entrées.
1

Créer l'index avec mapping précis

Définir les types de champs pour optimiser la recherche et le filtrage.

Création index armes-fr
PUT /armes-fr
{
  "settings": {
    "number_of_shards": 3,
    "number_of_replicas": 1
  },
  "mappings": {
    "properties": {
      "nom":         { "type": "text",    "fields": { "keyword": { "type": "keyword" } } },
      "calibre":      { "type": "keyword" },
      "type":          { "type": "keyword" },
      "pays_origine": { "type": "keyword" },
      "fabricant":    { "type": "keyword" },
      "annee":         { "type": "integer" },
      "en_service":   { "type": "boolean" },
      "classifie":    { "type": "boolean" },
      "portee_m":     { "type": "integer" },
      "poids_kg":     { "type": "float" },
      "description":  { "type": "text",    "analyzer": "french" },
      "tags":          { "type": "keyword" },
      "ingested_at":  { "type": "date" }
    }
  }
}
2

Bulk indexation 4 armes

3

Recherche multi-critères avec agrégations

Requête typique d'un opérateur cherchant une arme en service avec statistiques.

Recherche + stats par type
GET /armes-fr/_search
{
  "query": {
    "bool": {
      "must": [
        { "multi_match": {
          "query": "fusil assaut moderne",
          "fields": ["nom^3", "description", "tags"]
        }}
      ],
      "filter": [
        { "term": { "en_service": true } },
        { "term": { "classifie": false } },
        { "range": { "annee": { "gte": 2000 } } }
      ]
    }
  },
  "sort": [
    { "_score": { "order": "desc" } },
    { "portee_m": { "order": "desc" } }
  ],
  "aggs": {
    "par_type": {
      "terms": { "field": "type", "size": 10 },
      "aggs": {
        "portee_max": { "max": { "field": "portee_m" } },
        "poids_moy": { "avg": { "field": "poids_kg" } }
      }

Requête ES|QL pour rapport analytique

Générer un rapport de synthèse sur les armes françaises en service.

Rapport ES|QL
POST /_query
{
  "query": """
    FROM armes-fr
    | WHERE en_service == true AND classifie == false
    | EVAL categorie = CASE(
        portee_m > 1000, "longue portée",
        portee_m > 200,  "portée intermédiaire",
        "courte portée"
      )
    | STATS
        total     = COUNT(*),
        poids_moy = AVG(poids_kg),
        portee_max = MAX(portee_m)
      BY categorie, type
    | SORT total DESC
  """
}
