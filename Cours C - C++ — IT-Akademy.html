Les bases : variables, types, opérateurs

C et C++ partagent la même syntaxe de base. On commence par ce que les deux langages ont en commun avant de voir leurs différences.

🏗️ Structure minimale d'un programme

En C et C++, chaque programme doit avoir une fonction main(). C'est le point d'entrée, le premier endroit où le compilateur commence à lire.

C
// Fichier : hello.c
#include <stdio.h>   // bibliothèque pour printf

int main() {
    printf("Bonjour IT-Akademy !\n");
    return 0;   // 0 = tout s'est bien passé
}
C++
// Fichier : hello.cpp
#include <iostream>  // bibliothèque pour cout

int main() {
    std::cout << "Bonjour IT-Akademy !" << std::endl;
    return 0;
}
💡

main() vs Java : Comme en Java, main() est le point de départ. Différence : en C/C++, toutes les classes/fichiers partagent un seul main(). En Java, chaque classe peut avoir le sien.

Fonctions

Une fonction = un bloc de code réutilisable avec un nom. Même concept qu'en Java, mais sans classes obligatoires en C.

🔧 Syntaxe de base

C/C++
// Déclaration : type_retour nom(paramètres)
int addition(int a, int b) {
    return a + b;
}

// void = ne retourne rien
void afficherBienvenue(char *nom) {
    printf("Bienvenue %s !\n", nom);
}

int main() {
    int resultat = addition(10, 5);
    printf("%d\n", resultat);  // 15
    afficherBienvenue("Tom");
    return 0;
}

🔀 Surcharge en C++ (pas en C !)

En C++, on peut avoir plusieurs fonctions avec le même nom mais des paramètres différents.

C++ uniquement
// Même nom, paramètres différents = surcharge
int max(int a, int b) {
    return (a > b) ? a : b;
}

double max(double a, double b) {
    return (a > b) ? a : b;
}

// C++ choisit automatiquement la bonne version
max(3, 5);        // appelle la version int
max(3.2, 5.1);    // appelle la version double

La mémoire — LE sujet clé en C

C'est ici que C devient différent de TOUS les autres langages. Tu gères la RAM toi-même. C'est puissant, dangereux, et c'est pour ça que les OS, virus et logiciels critiques sont écrits en C.

🧠 Stack vs Heap — la distinction fondamentale

Stack (Pile) 📌Heap (Tas) 📦
GestionAutomatiqueManuelle (malloc / free)
VitesseUltra rapidePlus lent
TailleLimitée (~8 Mo)Grande (RAM dispo)
Durée de vieFin de la fonctionJusqu'au free()

C — Stack vs Heap
#include <stdio.h>
#include <stdlib.h>  // pour malloc et free

int main() {

    // ═══ STACK (automatique) ═══
    int a = 42;          // créé et détruit automatiquement

    // ═══ HEAP (manuel) ═══
    int *p = malloc(sizeof(int));  // réserve 4 octets

    if (p == NULL) {          // TOUJOURS vérifier
        printf("Erreur mémoire\n");
        return 1;
    }

    *p = 99;                 // écrire dans la mémoire allouée
    printf("Valeur : %d\n", *p);

    free(p);                 // OBLIGATOIRE sinon fuite mémoire
    p = NULL;               // bonne pratique : mettre à NULL après free

    return 0;
}

☠️ Les 4 erreurs classiques

💣

1. Buffer Overflow — écrire en dehors d'un tableau → faille exploitable

char buf[5]; strcpy(buf, "trop_long"); → BOOM

🔁

2. Double free — libérer deux fois la même mémoire → crash

free(p); free(p); → comportement indéfini

👻

3. Use After Free — utiliser la mémoire après free() → données corrompues ou exploitation

free(p); *p = 10; → dangereux

💧

4. Memory Leak — oublier free() → la mémoire ne se libère jamais

malloc(1000); // sans free → ton programme grossit indéfiniment

Les pointeurs — le GPS de la mémoire

Les pointeurs sont LA chose qui fait peur en C. En réalité c'est simple : un pointeur est une variable qui stocke une adresse mémoire. Rien de plus.

🧭 L'analogie GPS

🗺️

p = l'adresse GPS 📍 (ex: 0x7ffe1234)  |  *p = ce qu'il y a à cette adresse 🏠

C — Pointeurs décryptés
int a = 42;
int *p;   // p est un pointeur vers un int (contient une adresse)

p = &a;   // &a = "l'adresse de a" → p pointe vers a

printf("Valeur de a : %d\n", a);     // 42
printf("Adresse de a : %p\n", p);    // 0x7ffe...
printf("Valeur via p : %d\n", *p);   // 42 (déréférencement)

*p = 100;  // modifie a à travers le pointeur
printf("a vaut maintenant : %d\n", a); // 100 !

📋 Récap des symboles

SymboleSignificationExemple
* (dans déclaration)Ce type est un pointeurint *p;
&L'adresse mémoire dep = &a;
* (après déclaration)Aller à l'adresse = déréférencer*p = 5;

🔗 Tableaux et pointeurs

C
int arr[3] = {10, 20, 30};
int *p = arr;  // arr = adresse du 1er élément

printf("%d\n", *p);       // 10 (1er élément)
printf("%d\n", *(p+1));   // 20 (2e élément)
printf("%d\n", *(p+2));   // 30 (3e élément)
EX 07 🔄 Swap de variables sans variable temporaire... ou presque

Crée une fonction swap(int *a, int *b) qui échange les valeurs de deux variables en utilisant des pointeurs.

  • La fonction doit modifier les variables originales (pas des copies)
  • Test : a = 3, b = 7 → après swap : a = 7, b = 3
💡

C'est exactement comment les algorithmes de tri modifient les tableaux en place.

🔓 Voir la solution
✅ Solution
C
#include <stdio.h>

void swap(int *a, int *b) {
    int temp = *a;  // sauvegarde a
    *a = *b;        // a prend la valeur de b
    *b = temp;     // b prend la valeur sauvegardée
}

int main() {
    int a = 3, b = 7;
    printf("Avant : a=%d b=%d\n", a, b);
    swap(&a, &b);  // on passe les adresses
    printf("Après : a=%d b=%d\n", a, b);
    return 0;
}

Compilation

Contrairement à Java (JVM) ou Python (interprété), C et C++ compilent directement en binaire natif. Résultat : un exécutable super rapide, sans rien d'autre nécessaire.

⚙️ Les commandes essentielles

bash
# Compiler un fichier C
gcc prog.c -o prog

# Compiler un fichier C++
g++ prog.cpp -o prog

# Lancer le programme
./prog           # Linux / Mac
.\prog.exe       # Windows

# Avec avertissements (recommandé)
gcc -Wall -Wextra prog.c -o prog

# Avec infos de débogage (pour gdb)
gcc -g prog.c -o prog

Mini-OS, terminal et concepts avancés

Maintenant que tu maîtrises les bases, voilà comment tout ça se connecte aux vraies applications système — et pourquoi C est roi dans ce domaine.

🖥️ Mini terminal en C

C — Mini Shell
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main() {
    char commande[256];

    printf("=== IT-Akademy Shell v0.1 ===\n");

    while (1) {
        printf("it-shell> ");
        fgets(commande, sizeof(commande), stdin);

        // Supprimer le \n final
        commande[strcspn(commande, "\n")] = 0;

        if (strcmp(commande, "exit") == 0) {
            printf("Bye!\n");
            break;
        }

        system(commande);  // exécute la commande OS
    }
    return 0;
}

⚙️ Kernel minimaliste (concept OS)

Un vrai kernel démarre sans aucune bibliothèque. Il écrit directement dans la mémoire vidéo du CPU.

C — Kernel baremetal
// ATTENTION : ce code tourne sans OS, directement sur CPU
// Compile avec : gcc -ffreestanding -o kernel kernel.c

void kernel_main() {
    // Mémoire vidéo VGA mode texte : adresse fixe 0xB8000
    char *video = (char*) 0xB8000;

    // Chaque caractère = 2 octets : char + couleur
    video[0] = 'I'; video[1] = 0x0F;  // blanc sur noir
    video[2] = 'T'; video[3] = 0x0F;
    video[4] = '-'; video[5] = 0x0F;
    video[6] = 'A'; video[7] = 0x0F;

    while(1); // boucle infinie (l'OS ne se "termine" pas)
}
🧠

Voilà pourquoi Linux, Windows NT, et macOS sont écrits en C : c'est le seul langage qui peut parler directement au matériel, sans intermédiaire. C = le langage de la machine.

EX 09 🗂️ Logger de commandes

Améliore le mini shell : au lieu d'exécuter les commandes, écris-les dans un fichier log.txt avec l'heure approximative (utilise un compteur). Affiche "Commande enregistrée" après chaque saisie.

🔓 Voir la solution
✅ Solution
C
#include <stdio.h>
#include <string.h>

int main() {
    char commande[256];
    int compteur = 0;

    FILE *log = fopen("log.txt", "w");
    if (!log) { printf("Erreur log\n"); return 1; }

    printf("=== Logger Shell ===\n");

    while (1) {
        printf("logger> ");
        fgets(commande, sizeof(commande), stdin);
        commande[strcspn(commande, "\n")] = 0;

        compteur++;
    }

    fclose(log);
    printf("%d commandes enregistrees dans log.txt\n", compteur);
    return 0;
}

EX 11 🔢 XOR Cipher basique

Le chiffrement XOR est l'un des plus simples — et il est réversible ! Chiffre puis déchiffre un texte avec une clé entière.

  • Pour chiffrer : texte[i] ^= cle (opérateur XOR)
  • Pour déchiffrer : même opération avec la même clé
  • Test avec "hello" et clé = 42
🔓 Voir la solution
✅ Solution
C
#include <stdio.h>
#include <string.h>

void xorCipher(char *texte, int cle) {
    for (int i = 0; i < strlen(texte); i++) {
        texte[i] ^= cle;  // XOR avec la clé
    }
}

int main() {
    char message[] = "hello";
    int cle = 42;

    printf("Original : %s\n", message);

    xorCipher(message, cle);
    printf("Chiffré  : %s\n", message);  // illisible

    xorCipher(message, cle);  // même opération = déchiffrement
    printf("Déchiffré: %s\n", message);  // hello à nouveau

    return 0;
}
EX 12 ⭐ 🏆 BONUS — Programme qui se copie lui-même

Concept de base des virus : un programme qui lit son propre code source et le copie ailleurs. Inoffensif, purement éducatif.

⚠️

Ce programme utilise __FILE__, une macro C qui contient le chemin du fichier source courant. Ne marche qu'à la compilation, pas sur le binaire final.

🔓 Voir la solution
✅ Solution
C
#include <stdio.h>

int main() {
    FILE *source = fopen(__FILE__, "r");  // lit CE fichier
    FILE *copie = fopen("copie.c", "w");

    if (!source || !copie) {
        printf("Erreur d'ouverture\n");
        return 1;
    }

    char c;
    while ((c = fgetc(source)) != EOF) {
        fputc(c, copie);
    }

    fclose(source);
    fclose(copie);

    printf("Code source copié dans copie.c\n");
    printf("(Principe de base de la réplication virale)\n");
    return 0;
}

Un vrai virus va plus loin : il cible des exécutables, modifie leur code binaire, et se lance sans que l'utilisateur le sache. Mais le principe de "lire et copier" reste le même.
