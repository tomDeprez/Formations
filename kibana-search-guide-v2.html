1 C'est quoi Discover ?

Discover est l'interface principale de Kibana pour explorer et rechercher vos données. C'est votre point d'entrée : vous choisissez un index (Data View), vous cherchez, vous filtrez, et vous visualisez les résultats sous forme de tableau et d'histogramme temporel.

2 Créer un Data View (obligatoire)

Avant de pouvoir chercher quoi que ce soit, vous devez créer un Data View (anciennement appelé "Index Pattern"). C'est ce qui dit à Kibana quels index Elasticsearch afficher.

⚠️
Sans Data View, pas de Discover

Si vous ouvrez Discover pour la première fois et que vous voyez "No data view", c'est normal — vous devez en créer un d'abord.

1

Aller dans Stack Management

Cliquez sur le ☰ menu hamburger en haut à gauche → descendez jusqu'à Management → cliquez sur Stack Management.

☰›Management›Stack Management
2

Ouvrir "Data Views"

Dans Stack Management, cherchez la section Kibana → cliquez sur Data Views.

Stack Management›Kibana›Data Views
3

Cliquer sur "Create data view"

Cliquez sur le bouton bleu "Create data view" en haut à droite.

4

Renseigner le pattern d'index

Dans le champ "Index pattern", tapez le pattern qui correspond à vos index. Kibana affiche en temps réel les index qui correspondent.

Exemples de patterns d'index

nginx-* → tous les index commençant par nginx

5

Sélectionner le champ timestamp

Dans le menu déroulant "Timestamp field", sélectionnez @timestamp (c'est le champ de date standard Elasticsearch). Cliquez ensuite sur "Save data view to Kibana".

✅ Vous pouvez maintenant aller dans Discover et voir vos données !

5 Gérer les colonnes affichées

Par défaut Discover affiche le champ _source (tout le document). Vous pouvez choisir quels champs afficher en colonnes.

1

Via la liste de champs (panneau gauche)

Dans le panneau gauche de Discover, vous avez la liste de tous les champs. Survolez un champ → cliquez sur le bouton "+ Add" qui apparaît. Le champ s'ajoute comme colonne dans le tableau de résultats.

Champs disponibles
int status + Add
kw source.ip + Add
txt message + Add
2

Réordonner et supprimer des colonnes

Dans les en-têtes du tableau, vous pouvez glisser-déposer les colonnes pour les réordonner. Cliquez sur le × à droite du nom de colonne pour la supprimer.

6 Changer la fenêtre temporelle

Kibana affiche toujours les données dans une plage de temps. Par défaut : "Last 15 minutes". Le sélecteur de temps est en haut à droite.

1

Cliquer sur le sélecteur de temps

🔵 KQL — Kibana Query Language

KQL — Le langage
par défaut de Kibana

KQL est le langage de recherche activé par défaut dans la barre de recherche de Discover. Simple, avec autocomplétion, idéal pour la majorité des recherches quotidiennes.

☰›Analytics › Discover›Barre de recherche — "KQL" activé par défaut

1 Activer KQL

Dans la barre de recherche de Discover, vérifiez que le badge KQL est visible à droite. Si vous voyez Lucene à la place, cliquez dessus et sélectionnez KQL.

2 Syntaxe complète

KQL — Référence complèteKQL
# ─── VALEURS EXACTES ──────────────────────────────────
status: 404                        # nombre
method: "GET"                      # texte exact
tls.established: true             # booléen

# ─── PLUSIEURS VALEURS (= IN) ────────────────────────
status: (400 OR 401 OR 403 OR 404)
event.category: ("network" OR "process")

# ─── WILDCARDS ───────────────────────────────────────
url.path: *admin*                 # contient admin
host.name: web-*                  # commence par web-
source.ip: 192.168.*              # sous-réseau

# ─── PLAGES NUMÉRIQUES ──────────────────────────────
status >= 400
status >= 400 AND status < 500
http.response.bytes > 1000000

# ─── DATES ───────────────────────────────────────────
@timestamp > "now-1h"
@timestamp > "now-24h" AND @timestamp < "now"
@timestamp >= "2024-01-15T14:00:00"

# ─── CHAMP EXISTANT / ABSENT ─────────────────────────
error.message: *                   # champ présent
NOT user.name: *                   # champ absent/null

# ─── OPÉRATEURS LOGIQUES ─────────────────────────────
status: 404 AND method: "POST"
status: 404 OR  status: 500
NOT tags: "_grokparsefailure"
(status: 401 OR status: 403) AND NOT source.ip: "10.*"

# ─── TEXTE LIBRE (full-text) ─────────────────────────
message: "authentication failure"   # phrase exacte
message: nginx error               # les deux mots
⌨️
Autocomplétion

Commencez à taper un nom de champ → Kibana propose les champs disponibles. Tapez : → il propose les valeurs les plus fréquentes. Utilisez les flèches ↑↓ pour naviguer.

🟢 Lucene Query Syntax

Lucene — Regex,
fuzzy et boosting

Plus puissant que KQL : regex, fuzzy search, proximity. S'active via le toggle dans la barre de recherche Discover.

1 Activer Lucene dans Kibana

1

Cliquer sur le badge "KQL"

Dans la barre de recherche de Discover, cliquez sur le badge KQL à droite de la barre.

2

Sélectionner "Lucene"

Un menu apparaît avec KQL et Lucene. Cliquez sur Lucene. Le badge devient Lucene.

2 Syntaxe Lucene

Lucene — RéférenceLUCENE
# ─── BASE ────────────────────────────────────────────
status:404
message:"connection refused"

# ─── WILDCARDS ───────────────────────────────────────
host:web-0?          # ? = 1 caractère
url:*admin*
host:prod-*

# ─── REGEX (entre //) ────────────────────────────────
url:/(sql|union|select)/i
source.ip:/185\.220\.[0-9]+\.[0-9]+/

# ─── FUZZY (correspondance approchée) ────────────────
message:authetication~    # matche "authentication"
message:nginx~2           # distance Levenshtein max 2

# ─── PLAGES ──────────────────────────────────────────
status:[400 TO 499]     # inclusif
status:{400 TO 500}     # exclusif
status:[400 TO *]        # ≥ 400

# ─── BOOSTING ────────────────────────────────────────
title:nginx^3 message:nginx  # titre 3x plus important

# ─── OPÉRATEURS ──────────────────────────────────────
status:404 AND host:prod-*
+status:404 -source:internal  # + = must, - = must not
ℹ️
Quand choisir Lucene plutôt que KQL ?

Utilisez Lucene quand vous avez besoin de regex (/pattern/), de fuzzy (~), ou de boosting (^). Pour tout le reste, KQL est plus lisible.

🟠 Query DSL

Query DSL — Contrôle
total via la Console

Le langage natif d'Elasticsearch en JSON. Utilisé dans la Dev Tools Console de Kibana pour des agrégations complexes, du scoring personnalisé, des nested queries.

1 Accéder à la Console

☰›Management›Dev Tools → Console

Tapez vos requêtes dans le panneau gauche → appuyez sur le ▶ bouton Play (ou Ctrl+Entrée) → les résultats apparaissent à droite.

2 Requêtes essentielles

Query DSL — FondamentauxDSL
// ─── RECHERCHE SIMPLE ─────────────────────────────
GET /nginx-*/_search
{
  "size": 10,
  "query": { "term": { "status": 404 } }
}

// ─── BOOL QUERY (la plus utilisée) ────────────────
GET /logs-*/_search
{
  "query": {
    "bool": {
      "must":    [{ "match": { "message": "Failed password" } }],
      "filter":  [{ "term": { "destination.port": 22 } }],
      "must_not":[{ "term": { "source.ip": "10.0.0.1" } }]
    }
  }
}

// ─── AGRÉGATIONS ──────────────────────────────────
GET /nginx-*/_search
{
  "size": 0,
  "aggs": {
    "par_status": { "terms": { "field": "status" } },
    "par_heure": {
      "date_histogram": {
        "field": "@timestamp",
        "calendar_interval": "1h"
      }
    }
  }
}
🟣 SQL

SQL dans Kibana —
Dev Tools Console

Interrogez Elasticsearch avec du SQL standard. Idéal pour les exports, reporting, et si vous connaissez déjà SQL. Via POST /_sql dans la Console.

1 Comment utiliser SQL dans Kibana

1

Ouvrir Dev Tools Console

☰›Management›Dev Tools → Console
2

Taper la requête SQL

Utilisez l'endpoint POST /_sql avec votre requête SQL dans le champ "query". Les noms d'index vont entre guillemets doubles.

3

Choisir le format de retour

Ajoutez ?format=txt pour un tableau lisible, ?format=csv pour un export.

SQL — Exemples pratiquesSQL
-- Recherche basique
POST /_sql?format=txt
{ "query": "SELECT source.ip, status, url.path FROM \"nginx-*\" WHERE status >= 400 ORDER BY @timestamp DESC LIMIT 20" }

-- Compter par code HTTP
POST /_sql?format=txt
{ "query": "SELECT status, COUNT(*) AS total FROM \"nginx-*\" GROUP BY status ORDER BY total DESC" }

-- Dernière heure avec LIKE
POST /_sql
{ "query": "SELECT * FROM \"logs-*\" WHERE @timestamp > NOW() - INTERVAL 1 HOUR AND url.path LIKE '%admin%'" }

-- Top IPs attaquantes
POST /_sql?format=txt
{ "query": "SELECT source.ip, COUNT(*) AS hits FROM \"nginx-*\" WHERE status IN (401,403,404) GROUP BY source.ip HAVING COUNT(*) > 50 ORDER BY hits DESC LIMIT 10" }
🩷 EQL — Event Query Language

EQL — Détecter des
séquences d'attaque

EQL est le langage des règles SIEM dans Kibana Security. Il permet de corréler des événements dans le temps et de détecter des enchaînements suspects.

1 Où utiliser EQL dans Kibana

1

Via Kibana Security → Timelines

☰›Security›Timelines›Onglet "Correlation"
2

Via Dev Tools Console (API)

Utilisez GET /index/_eql/search directement dans la Console.

EQL — Syntaxe et séquencesEQL
// ─── BASE : catégorie where conditions ──────────────
process where process.name == "cmd.exe"
network where destination.port == 22 and event.outcome == "failure"

// ─── SÉQUENCE TEMPORELLE ─────────────────────────
// Détecter brute force SSH puis connexion réussie
sequence by source.ip with maxspan=10m
  [authentication where event.outcome == "failure"] with runs=5
  [authentication where event.outcome == "success"]

// Détecter Word qui lance cmd.exe (macro malveillante)
sequence by host.id with maxspan=30s
  [process where process.name == "winword.exe"]
  [process where process.name in ("cmd.exe", "powershell.exe")]

// Via API
GET /logs-endpoint.*/_eql/search
{ "query": "process where process.name == 'cmd.exe'", "size": 10 }
🩵 ES|QL

ES|QL — Le nouveau
langage pipe de Kibana

Disponible depuis ES 8.11 directement dans Discover. Syntaxe pipe (comme Unix), transformations inline, GROK intégré. L'avenir des requêtes analytiques dans Kibana.

1 Activer ES|QL dans Discover

1

Ouvrir Discover

☰›Analytics›Discover
2

Cliquer sur le sélecteur de langue

En haut à gauche de Discover, cliquez sur le sélecteur de Data View. À côté, vous verrez le sélecteur de langue. Choisissez ES|QL.

Discover — sélection du langage
KQL
Lucene
ES|QL ✓
FROM nginx-* | WHERE status >= 400 | LIMIT 50
3

Écrire la requête et lancer

La barre de recherche change de style. Tapez votre requête ES|QL → appuyez sur ↵ Entrée. Les résultats s'affichent en tableau.

2 Requêtes ES|QL dans Discover

ES|QL — Exemples DiscoverES|QL
// ─── BASIQUE ─────────────────────────────────────
FROM nginx-*
  | WHERE status >= 400
  | KEEP @timestamp, source.ip, status, url.path
  | SORT @timestamp DESC
  | LIMIT 50

// ─── STATS : top IPs avec erreurs ────────────────
FROM nginx-*
  | WHERE status >= 400 AND @timestamp >= "now-24h"
  | STATS nb = COUNT(*) BY source.ip
  | SORT nb DESC
  | LIMIT 10

// ─── EVAL : classer les codes HTTP ───────────────
FROM nginx-*
  | EVAL type = CASE(
      status >= 500, "Erreur serveur",
      status >= 400, "Erreur client",
      "OK"
    )
  | STATS n = COUNT(*) BY type
  | SORT n DESC

// ─── GROK inline sans pipeline Logstash ──────────
FROM logs-raw
  | GROK message "%{IP:src_ip} - %{WORD:method} %{NOTSPACE:uri}"
  | WHERE method == "POST"
  | KEEP src_ip, method, uri
  | LIMIT 50
💡
ES|QL dans la Console aussi

En plus de Discover, vous pouvez utiliser ES|QL via POST /_query dans la Dev Tools Console : {"query": "FROM nginx-* | WHERE status >= 400 | LIMIT 10"}
